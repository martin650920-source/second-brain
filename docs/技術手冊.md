# 第二大腦 — 技術手冊

---

## 一、整體架構概念

### 比喻

把第二大腦想成一個「個人 AI 作業系統」：

```
你的電腦
└── Google Drive（備份 + 跨裝置同步）
    └── second-brain/（唯一本體）
            ↓ setup 腳本建立捷徑（symlink）
        AI 工具啟動時自動讀取
```

**核心設計原則：**
- **一份本體** — 所有設定只寫一次，多個 AI 工具共用
- **按需載入** — 只燒必要的 token，不用的不載
- **公私分離** — 骨架進 Git 公開，私密內容留本機

---

## 二、三層載入架構

```
第一層：全域層（每次都載）
    core/profile.md       ← 你是誰、偏好
    core/meta-rules.md    ← 衝突時誰優先

        +

第二層：情境層（你選才載）
    contexts/work/        ← 工作：只有公司工具
    contexts/life/        ← 生活：只有個人工具

        +

第三層：專案層（自動偵測才載）
    projects/nagra-tntsat.md
    projects/android-aosp.md
```

**優先序（有衝突時）：專案層 > 情境層 > 全域層**

---

## 三、Token 消耗說明

```
session 啟動
    ↓
CLAUDE.md（5行）           ← 固定消耗，極小
    ↓
SKILL.md（~150行）         ← 固定消耗，中型
    ↓
profile.md（~600字）       ← 固定消耗，小
meta-rules.md（~400字）    ← 固定消耗，小
    ↓
contexts/work/*.md         ← 選 work 才燒
    ↓
projects/xxx.md            ← 偵測到才燒
    ↓
其他所有檔案               ← 0，永遠不自動載入
```

**設計紀律：**
- 規則只放「指令」，不放「內容」
- 內容放獨立 md，由 SKILL.md 按需讀取
- `profile.md` 強制 ≤ 1500 字

---

## 四、Secure Mode 運作方式

```
~/.second-brain/.secure-mode 檔案存在
    ↓
context-loader 偵測到
    ↓
嚴格執行 MCP 白名單
    ↓
work 情境：只有 mcp.json 列出的公司工具存在
life 情境：只有 mcp.json 列出的個人工具存在
    ↓
AI 被要求使用白名單外的工具 → 拒絕並說明原因
```

## 七、Session 啟動完整流程

```
1. Claude Code 啟動
        ↓
2. 讀 ~/.claude/CLAUDE.md
   → symlink 指向 adapters/claude/CLAUDE.md
   → 內容：「去讀 SKILL.md」
        ↓
3. 讀 skills/context-loader/SKILL.md
        ↓
4. 確認 ~/.second-brain/.secure-mode 是否存在
        ↓
5. 讀 core/profile.md（你是誰）
   讀 core/meta-rules.md（規則優先序）
        ↓
6. 詢問情境選擇
   選 work → 讀 contexts/work/context.md + rules.md
             讀 contexts/work/mcp.json → 列出可用工具
   選 life → 讀 contexts/life/context.md
             讀 contexts/life/mcp.json
   選 none → 跳過
        ↓
7. 掃 cwd 偵測 Marker Files
   有符合 → 確認，讀 projects/<name>.md
   沒符合 → 列清單讓你選，或跳過
        ↓
8. 輸出載入摘要，等待指令
```

## 八、未來擴展路徑

### 第二階段：知識圖譜（LLM-Wiki）

```
新增：wiki/ + raw/

工作流程：
  你丟資料到 raw/ → 叫 AI「消化這份，更新 wiki/」
  AI 產出變更建議 → 你在 Obsidian 確認 → 寫入 wiki/

wiki/ 結構：
  wiki/index.md              ← AI 維護的目錄
  wiki/log.md                ← 操作紀錄
  wiki/work/<主題>.md        ← 工作知識頁
  wiki/tech/<主題>.md        ← 技術知識頁
```

### 第三階段：維護自動化

```
新增：tools/health-check.sh（純腳本，零 token）

功能：
  - 孤立頁偵測（grep）
  - 斷鏈檢查（grep）
  - 過時頁偵測（find + mtime）
  - 知識庫大小統計

定期（每月）：
  - AI 對照 raw/ 重新驗證 wiki/ 內容
  - 封存 90 天未讀的過時頁
```

### 第四階段：搜尋引擎

```
當 wiki/ 超過 100 頁時：
  加入本地搜尋（BM25 + 向量）
  Claude Code 可透過 MCP 呼叫搜尋
  資料永不離開本機
```

### 新增情境（隨時可做）

```
新增 research/ 情境：
  contexts/research/context.md
  contexts/research/mcp.json
→ 適合深度研究、閱讀論文時使用
```

### Schema 版本化（長期）

```
當 SKILL.md 規則演化時：
  在 meta-rules.md 加版號
  舊頁面 migrate 到新格式
  避免規則改了但舊知識頁還用舊格式
```

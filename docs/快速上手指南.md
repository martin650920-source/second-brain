## 🖥️ 首次安裝（Windows）

**開 PowerShell（系統管理員，或已開啟開發者模式）**
```powershell
cd "D:\GOOGLE_DRIVE_SYNC\ai-workspace"
pwsh -ExecutionPolicy Bypass -File setup\setup-windows.ps1
```

**腳本實際會建立的 7 條 symlink：**
```
~/.ai-workspace                 → Google Drive 的 ai-workspace/
~/.claude/CLAUDE.md             → adapters/claude/CLAUDE.md
~/.claude/skills/<name>         → skills/global/<name>/（每個 skill 各一條）
~/.claude/statusline.sh         → config/statusline.sh
~/.gemini/GEMINI.md             → adapters/gemini/GEMINI.md
~/.codex/AGENTS.md              → adapters/codex/AGENTS.md
<Google Drive 根目錄>/CLAUDE.md → ai-workspace/projects/google-drive-sync.md
```
最後一條是「Google Drive 同步根目錄」本身的專案層 CLAUDE.md（讓 AI 認得這個 vault 的結構），跟工作專案是分開的機制。

**安裝過程會問你：**
- 偵測到舊版 symlink？→ 輸入 `Y` 更新
- 現有路徑要備份還是忽略？→ 看提示選擇

**確認安裝成功：** 腳本結尾的驗證區塊會列出上面 7 條 symlink 逐一打勾，`[MISSING]` 代表沒建成功。

## 🔗 首次安裝（SSH Remote）

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/martin650920-source/ai-workspace/main/setup/setup-ssh.sh)" _ git@github.com:martin650920-source/ai-workspace.git
```

或先 clone 再跑：
```bash
git clone git@github.com:martin650920-source/ai-workspace.git /tmp/aw-setup
bash /tmp/aw-setup/setup/setup-ssh.sh git@github.com:martin650920-source/ai-workspace.git
```

**SSH 跟 Windows/WSL 的差異：**
- 沒有 Google Drive 可掛載，改成 `git clone` 到 `~/.ai-workspace-git`，`~/.ai-workspace` symlink 指向這個 clone
- 不會建立「Google Drive 根目錄 CLAUDE.md」那條 symlink（沒有 vault 概念）
- 安裝最後會問要不要設定**每日 09:00 自動 `git pull`**（cron job，log 寫在 `~/.ai-workspace-pull.log`）

## ✅ 安裝後驗收

重啟 Claude Code，開新 session，應該會看到：

```
## Session Context Loaded
- Global : rules/global.md ✓
- Project: projects/<name>.md ✓  |  rules: [有/無]  |  mem: [有/無]
Ready. What are we working on today?
```

**如果沒看到** → 在 Claude Code 輸入「載入 context」手動觸發

**如果在一個還沒接上 ai-workspace 的目錄開 session**（cwd 沒有 CLAUDE.md、有但不是 symlink，或是 symlink 但指向的專案檔已經不存在／斷鍊），會先跳出選單：
```
1. Link project（建立/連結專案本體到 ai-workspace）
2. Remove project（清理 ai-workspace 內某專案的殘留檔案）
```
選 `1` 之後才會看到 New / 既有專案清單 / Skip 的選單。

**已經連結過的目錄**：
- Session 剛開始自動載入時 → 不會跳選單，直接零打擾載入
- 你事後在同一個 session 裡手動說「載入 context」或打 `/context-loader` → **還是會跳出選單**（因為主動再叫一次，通常代表你想做點什麼，例如換連結或移除）

```
開 Claude Code
    ↓
context-loader 自動載入 rules/global.md
    ↓
cwd 已連結專案？
    ├─ 是 → 直接載入該專案（零打擾）
    └─ 否 → 跳 Link/Remove 選單，選擇後載入（或 Skip）

開始工作

結束時（有重要知識想留存）
    ↓
叫 AI「幫我記住 XXX」→ AI 會判斷寫進 rules 或 mem
```

## ➕ 新增工作專案（現在怎麼做）

**推薦方式：在專案目錄直接開 Claude Code**，session 開始時 context-loader 會自動跳選單：
```
1. Link project
    → 1. New
    → 輸入新專案名稱
    → 若這個目錄本來就有真實的 CLAUDE.md（已寫好內容）→ 直接把內容搬進 ai-workspace 保留下來
    → 若沒有 → 觸發 /init-project-md 分析 codebase 自動產生
    → 自動建立 ai-workspace/projects/<name>.md + 目錄下的 symlink
```

**手動方式**（例如本體已經先寫好、要接到某台機器的專案目錄）：
```bash
# 1. 把 CLAUDE.md 內容放到 ai-workspace/projects/<name>.md
# 2. 到 ai-workspace 目錄執行：
bash ~/.ai-workspace/setup/sync.sh link-project <專案路徑> <name>
```
會同時建立 `<專案路徑>/CLAUDE.md`（symlink）跟 `<專案路徑>/CLAUDE.local.md`（自動產生的 rules/mem import stub）。

**若 `<專案路徑>/CLAUDE.md` 已經是一份真實檔案**（不是 symlink，例如原本就有內容）→ 上面這行指令**會直接拒絕並報錯**，不會誤覆蓋，需要你確認內容可以捨棄後，重跑並加上：
```bash
bash ~/.ai-workspace/setup/sync.sh link-project <專案路徑> <name> --backup   # 先備份成 CLAUDE.md.bak-<日期> 再取代
bash ~/.ai-workspace/setup/sync.sh link-project <專案路徑> <name> --force    # 直接覆蓋，不備份
```

> 舊版需要手動編輯 `.gitignore` 加專案代號、編輯 `context-loader/SKILL.md` 加偵測規則兩個步驟，現在都不用了：
> - `.gitignore` 已改成不限制 `projects/*.md`（repo 目前是 Public，機密性由自己把關）
> - `context-loader` 的 Step 3 改用互動選單直接讀 `projects/` 目錄內容，不需要維護偵測表格

**新增完成後記得同步：**
```bash
bash ~/.ai-workspace/setup/sync.sh push
```

## ➖ 移除工作專案

在任一目錄開 Claude Code（不用一定要在該專案目錄下），context-loader 選單選 `2. Remove project`：
```
選要移除的專案 → 逐一詢問是否刪除：
  - projects/<name>.md
  - rules/projects/<name>/
  - mem/projects/<name>.md
  - skills/projects/<name>/（若有）
若目前所在目錄剛好連結到這個專案 → 一併問要不要順手清掉這個目錄的連結
```
每一項都會單獨確認，你可以中途選不刪、自己先備份。確認完，AI 會執行**一次**指令實際刪除：
```bash
bash ~/.ai-workspace/setup/sync.sh remove-project <name> [--paths=projects,rules,mem,skills] [--unlink-cwd]
```
（不帶 `--paths` 預設四類全刪；`--unlink-cwd` 會順便清掉目前目錄的 `CLAUDE.md` / `CLAUDE.local.md`，前提是它們剛好連結到這個專案）

**清完記得 `sync.sh push`**，另外**其他**主機專案目錄下殘留的 `CLAUDE.md` symlink 不會自動清除，需要自己到那些主機上手動刪。

## 🔄 保持最新

```bash
# SSH / WSL / Windows（Git Bash，含 symlink 校驗，已實測可用）：
bash ~/.ai-workspace/setup/sync.sh

# 上傳自己的變更：
bash ~/.ai-workspace/setup/sync.sh push
```

> Windows 上要用 **Git Bash**（不是 cmd.exe / PowerShell）：開始選單搜尋「Git Bash」，
> 或在檔案總管空白處右鍵「Open Git Bash here」。
>
> 若真的沒有 Git Bash 只能用 PowerShell，退而求其次純 pull（不含 symlink 校驗）：
> ```powershell
> cd D:\GOOGLE_DRIVE_SYNC\ai-workspace
> git pull
> ```

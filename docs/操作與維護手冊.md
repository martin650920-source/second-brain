## 一、安裝說明

### 三種環境的安裝方式

| 環境 | 腳本 | 說明 |
|---|---|---|
| Windows | `setup\setup-windows.ps1` | 需系統管理員身份，或開啟開發者模式 |
| WSL | `setup/setup-wsl.sh` | D: 磁碟需已掛載到 `/mnt/d/` |
| SSH Remote | `setup/setup-ssh.sh <git-url>` | 自動 clone 到 `~/.ai-workspace-git` + 設定 symlink，可選 cron 每日自動 pull |

這三支腳本只負責**首次 bootstrap**（本機還沒有 clone/掛載時）。日常更新請改用 `setup/sync.sh`（見第三節）。

### 安裝腳本做了什麼

**Windows / WSL（掛載 Google Drive，7 條 symlink）：**
```
1. 建立 ~/.ai-workspace → Google Drive 內的 ai-workspace/ 資料夾
2. 建立 ~/.claude/CLAUDE.md → adapters/claude/CLAUDE.md
3. 建立 ~/.claude/skills/<name> → skills/global/<name>/ （每個 skill 各一條）
4. 建立 ~/.claude/statusline.sh → config/statusline.sh
5. 建立 ~/.gemini/GEMINI.md → adapters/gemini/GEMINI.md
6. 建立 ~/.codex/AGENTS.md  → adapters/codex/AGENTS.md
7. 建立 <Google Drive 根目錄>/CLAUDE.md → ai-workspace/projects/google-drive-sync.md
   （這條是 vault 本身的專案層，跟工作專案分開）
```

**SSH Remote（沒有 Google Drive，git clone 版本，6 條 symlink）：**
```
1. git clone <repo> → ~/.ai-workspace-git（之後 pull 更新這份）
2. 建立 ~/.ai-workspace → ~/.ai-workspace-git
3~7. 同上 Windows/WSL 的 2~6（CLAUDE.md / skills / statusline / GEMINI.md / AGENTS.md）
   （不會建立第 7 條「Google Drive 根目錄 CLAUDE.md」，SSH 沒有 vault 概念）
8.（互動選填）設定每日 09:00 自動 git pull 的 cron job
```

### 從舊版遷移（ai_refrence / second-brain）

腳本會自動偵測舊版 symlink（指向 `ai-context` / `ai_refrence` / `second-brain`），提示你更新。你只需要：

1. 把 Google Drive 資料夾改名為 `ai-workspace`
2. 跑新的 setup 腳本
3. 看到 `[MIGRATE]` 提示時輸入 `Y`

> 若要把正式在用的 `second-brain` clone 換成這份新架構，還牽涉 GitHub repo 改名與多主機協調，
> 詳見根目錄 `ai-workspace-design.md` 開頭的改名 Checklist，不是單跑一次腳本就能完成。

## 二、日常使用

### 每次開啟 AI 工具

```
Claude Code / Gemini CLI 啟動
    ↓
自動讀 CLAUDE.md（bootstrap 指令）
    ↓
自動執行 context-loader
    ↓
自動載入 rules/global.md
    ↓
判斷 cwd 是否已連結專案：
  已連結（symlink 且目標存在）：
    session 剛開始自動載入 → 直接載入，不打擾
    事後手動說「載入 context」/ `/context-loader` → 仍會跳 Link/Remove 選單
  已連結但目標已被刪除（斷鍊）→ 提示連結失效，進選單處理
  未連結 → 跳 Link / Remove project 選單
    ↓
開始工作
```

### 讓知識留下來

```
工作中遇到新的解法、踩到坑 →
叫 AI：「幫我記住今天發現的 XXX 問題和解法」
→ AI 依「記憶寫入規則」判斷寫進 mem/projects/xxx.md 還是 mem/global.md

讀了有用的文章 →
丟到 raw/ 資料夾，叫 AI：「消化這份資料，更新到 wiki/」（第二階段）
```

## 三、維護工作

### 每週建議做

```bash
# 確認沒有過期資訊
打開 projects/<當前專案>.md，叫 AI 檢查：
「這份文件有沒有過時的內容？」

# 骨架同步到 Git（WSL/SSH，推薦用 sync.sh，含 symlink 校驗）
bash ~/.ai-workspace/setup/sync.sh push
```

### 新增工作專案（推薦：在專案目錄開 session，走互動選單）

```
1. cd 到專案目錄，開 Claude Code
2. context-loader 偵測到 cwd 沒接上 ai-workspace，跳選單：
   1. Link project
3. 選 New → 輸入新專案名稱
   - 這個目錄本來就有真實 CLAUDE.md（已寫好內容）→ 內容直接搬進 ai-workspace 保留
   - 沒有的話 → 觸發 /init-project-md，分析 codebase 自動產生
4. 自動完成：ai-workspace/projects/<name>.md 本體 + 目錄下 symlink
5. bash ~/.ai-workspace/setup/sync.sh push   ← 記得同步給其他主機
```

**手動方式**（本體已經先寫好、要接到某台機器的專案目錄時用）：
```bash
# ai-workspace/projects/<name>.md 已存在的前提下
bash ~/.ai-workspace/setup/sync.sh link-project <專案路徑> <name>
```
若 `<專案路徑>/CLAUDE.md` 已經是真實檔案（非 symlink）→ **指令會直接拒絕並報錯**，避免誤覆蓋。
確認內容可以捨棄後，重跑並加上 `--backup`（備份成 `CLAUDE.md.bak-<日期>` 後取代）或 `--force`（直接覆蓋）。

會同時建立 `<專案路徑>/CLAUDE.md`（symlink）跟 `<專案路徑>/CLAUDE.local.md`（自動產生的 rules/mem import stub，內容類似）：
```markdown
@~/.ai-workspace/rules/global.md
@~/.ai-workspace/rules/projects/<name>/general.md   ← 若該專案有 project rule 才會加這行
@~/.ai-workspace/mem/global.md
@~/.ai-workspace/mem/projects/<name>.md             ← 若該專案有 mem 才會加這行
```

> **不用再手動維護的兩件事**（舊版需要，現在都自動化了）：
> - ~~編輯 `.gitignore` 加專案代號~~ → 現在 `projects/*.md` 預設都能進 git，不用逐一列
> - ~~編輯 `context-loader/SKILL.md` 加偵測規則~~ → Step 3 改用互動選單，直接讀 `projects/` 目錄現有檔案

### 移除工作專案

任一目錄開 Claude Code，context-loader 選單選 `2. Remove project`：
```
1. 選要移除的專案 <name>
2. 逐一詢問是否刪除（可以中途說不刪，自己先備份）：
   - ai-workspace/projects/<name>.md
   - ai-workspace/rules/projects/<name>/
   - ai-workspace/mem/projects/<name>.md
   - ai-workspace/skills/projects/<name>/（若有）
3. 若目前所在目錄剛好連結到這個專案 → 一併問要不要順手清掉這個目錄的連結
4. 確認完，實際執行一次：
   bash ~/.ai-workspace/setup/sync.sh remove-project <name> [--paths=projects,rules,mem,skills] [--unlink-cwd]
5. bash ~/.ai-workspace/setup/sync.sh push   ← 記得同步
```
`sync.sh remove-project` 本身不會再互動跳確認（旗標驅動、一次執行），對話裡的逐項確認已經在第 2、3 步問完；
不帶 `--paths` 預設四類全刪，`--unlink-cwd` 只會清掉「目前所在目錄」剛好連結到這個專案的 `CLAUDE.md`/`CLAUDE.local.md`。

**注意：** **其他**主機專案目錄下殘留的 `CLAUDE.md` / `CLAUDE.local.md` symlink 不在 ai-workspace 管轄範圍，不會自動清除，需要自己到有連結過的主機上手動 `rm`。

### 新增 MCP 工具

```bash
# 編輯架構設定（引用環境變數，不含真實值）
編輯 ~/.ai-workspace/mcp/settings.json

# 真實憑證放家目錄，不進 ai-workspace repo
編輯 ~/.mcp.env
# 參考 ~/.ai-workspace/mcp/.env.example 的欄位格式
```

### 新增/修改 AI 規則

```bash
# 全域規則（所有專案都套用）
編輯 rules/global.md

# 特定專案全域規則
編輯 rules/projects/<專案名>/general.md

# 特定專案、只在特定路徑生效的規則
編輯 rules/projects/<專案名>/<主題名>.md（含 paths: frontmatter）
```

## 四、換新電腦

### Windows

```
1. 安裝 Google Drive 並等同步完成
2. 以系統管理員（或已開發者模式）開 PowerShell
3. pwsh -ExecutionPolicy Bypass -File "D:\GOOGLE_DRIVE_SYNC\ai-workspace\setup\setup-windows.ps1"
4. 補回 ~/.mcp.env（從舊電腦帶過來，不在 Google Drive 上）
```

### WSL / SSH

```bash
# WSL
bash /mnt/d/GOOGLE_DRIVE_SYNC/ai-workspace/setup/setup-wsl.sh

# SSH
bash setup-ssh.sh git@github.com:martin650920-source/ai-workspace.git
```

## 六、日後補強計畫（第二階段）

以下是確定要做、但刻意延後的功能，等第一階段穩定後再加：

### wiki/ 知識圖譜

```
raw/                ← 你丟原始資料進來
    ↓ 叫 AI 消化
wiki/index.md       ← AI 維護的知識目錄
wiki/<主題>/        ← AI 整理的知識頁，交叉連結
```

**Obsidian 整合：**
- 用 Obsidian 打開 `~/.ai-workspace/` 資料夾
- wiki/ 的連結圖自動呈現
- AI 寫，Obsidian 看

**AI 寫入規則（防崩壞）：**
- AI 只能「提出建議」，不能直接改 wiki
- 你在 Obsidian 看到建議後，確認才寫入

### 出處追溯

```
每個 wiki 頁面加上：
  source: raw/specs/xxx.pdf
  last_reviewed: 2026-07-01
```

讓 AI 定期回去驗證內容是否還準確。

### 決策紀錄

```
wiki/decisions/
    └── 2026-07-01-為何選A不選B.md
```

防止 AI 重複提出已否決的方案。

### 健康檢查指令

```bash
# 一個指令看整體狀態（腳本，不消耗 token）
bash ~/.ai-workspace/tools/health-check.sh

# 輸出：
# 總頁數: 42
# 孤立頁: 3
# 過時頁（>90天未更新）: 7
# 斷掉的連結: 0
```

### 知識庫搜尋

當 `wiki/` 大到 `index.md` 撐不住時，加入本地搜尋引擎（純本機，資料不外流）。

### 情境層（若日後需要）

舊版 work/life 情境 + Secure Mode 已整套移除。若之後真的需要工作/生活資產隔離，
要重新設計它跟現有 `rules/` 分層的關係，而非直接照搬舊版本。

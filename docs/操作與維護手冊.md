## 一、安裝說明

### 三種環境的安裝方式

| 環境 | 腳本 | 說明 |
|---|---|---|
| Windows | `setup\setup-windows.ps1` | 需系統管理員身份 |
| WSL | `setup/setup-wsl.sh` | D: 磁碟需已掛載 |
| SSH Remote | `setup/setup-ssh.sh <git-url>` | 自動 clone + 設定 |

### 安裝腳本做了什麼

```
1. 建立 ~/.second-brain → Google Drive 資料夾 的捷徑（symlink）
2. 建立 ~/.claude/CLAUDE.md → adapters/claude/CLAUDE.md
3. 建立 ~/.gemini/GEMINI.md → adapters/gemini/GEMINI.md
4. 建立 ~/.codex/AGENTS.md  → adapters/codex/AGENTS.md
5. 建立 ~/.claude/skills/<name> → skills/<name>/ （每個 skill 各一條）
6. 詢問是否啟用 Secure Mode
```

### 從舊版（ai_refrence）遷移

腳本會自動偵測舊版 symlink（指向 `ai-context` / `ai_refrence`），提示你更新。你只需要：

1. 把 Google Drive 資料夾改名為 `second-brain`
2. 跑新的 setup 腳本
3. 看到 `[MIGRATE]` 提示時輸入 `Y`

## 二、日常使用

### 每次開啟 AI 工具

```
Claude Code / Gemini CLI 啟動
    ↓
自動讀 CLAUDE.md（5行啟動指令）
    ↓
自動執行 context-loader
    ↓
你選情境（work / life / none）
    ↓
自動偵測專案，確認後載入
    ↓
開始工作
```

### 情境選擇建議

| 你要做什麼 | 選哪個情境 |
|---|---|
| 工作、寫程式、處理公司事務 | `work` |
| 生活咨詢、個人規劃、聊天 | `life` |
| 快速問答、不需要個人背景 | `none` |

### 讓知識留下來

```
工作中遇到新的解法、踩到坑 →
叫 AI：「幫我更新 projects/xxx.md，記錄今天發現的 XXX 問題和解法」

讀了有用的文章 →
丟到 raw/ 資料夾，叫 AI：「消化這份資料，更新到 wiki/」（第二階段）
```

## 三、維護工作

### 每週建議做

```bash
# 確認沒有過期資訊
打開 projects/<當前專案>.md，叫 AI 檢查：
「這份文件有沒有過時的內容？」

# 骨架同步到 Git
git -C ~/.second-brain add -A
git -C ~/.second-brain commit -m "update: ..."
git -C ~/.second-brain push
```

### 新增工作專案

```bash
# 1. 複製範本
cp ~/.second-brain/projects/_template.md \
   ~/.second-brain/projects/新專案.md

# 2. 在 .gitignore 加入（公司 IP 保護）
echo "/projects/新專案.md" >> ~/.second-brain/.gitignore

# 3. 在 context-loader 加偵測規則
# 編輯 skills/context-loader/SKILL.md
# 在 Step 5 表格加一行
```

### 新增 MCP 工具

```bash
# 工作情境加工具
編輯 contexts/work/mcp.json
# 參考 contexts/work/mcp.template.json 的格式

# 生活情境加工具
編輯 contexts/life/mcp.json
# 參考 contexts/life/mcp.template.json 的格式
```

### 新增/修改 AI 規則

```bash
# 全域規則（所有情境都套用）
編輯 core/profile.md

# 工作專用規則
編輯 contexts/work/rules.md

# 工作背景資訊
編輯 contexts/work/context.md
```

### Secure Mode 開關

```bash
# 開啟
touch ~/.second-brain/.secure-mode

# 關閉
rm ~/.second-brain/.secure-mode
```

## 四、換新電腦

### Windows

```
1. 安裝 Google Drive 並等同步完成
2. 以系統管理員開 PowerShell
3. pwsh -ExecutionPolicy Bypass -File "D:\GOOGLE_DRIVE_SYNC\second-brain\setup\setup-windows.ps1"
4. 補回 secrets/.env（從舊電腦帶過來，不在 Google Drive 上）
```

### WSL / SSH

```bash
# WSL
bash /mnt/d/GOOGLE_DRIVE_SYNC/second-brain/setup/setup-wsl.sh

# SSH
bash setup-ssh.sh git@github.com:martin650920-source/second-brain.git
```

## 五、日後補強計畫（第二階段）

以下是確定要做、但刻意延後的功能，等第一階段穩定後再加：

### wiki/ 知識圖譜

```
raw/                ← 你丟原始資料進來
    ↓ 叫 AI 消化
wiki/index.md       ← AI 維護的知識目錄
wiki/<主題>/        ← AI 整理的知識頁，交叉連結
```

**Obsidian 整合：**
- 用 Obsidian 打開 `~/.second-brain/` 資料夾
- wiki/ 的連結圖自動呈現
- AI 寫，Obsidian 看

**AI 寫入規則（防崩壞）：**
- AI 只能「提出建議」，不能直接改 wiki
- 你在 Obsidian 看到建議後，確認才寫入

### 出處追溯

```
每個 wiki 頁面加上：
  source: raw/specs/xxx.pdf
  last_reviewed: 2026-07-01
```

讓 AI 定期回去驗證內容是否還準確。

### 決策紀錄

```
wiki/decisions/
    └── 2026-07-01-為何選A不選B.md
```

防止 AI 重複提出已否決的方案。

### 健康檢查指令

```bash
# 一個指令看整體狀態（腳本，不消耗 token）
bash ~/.second-brain/tools/health-check.sh

# 輸出：
# 總頁數: 42
# 孤立頁: 3
# 過時頁（>90天未更新）: 7
# 斷掉的連結: 0
```

### 知識庫搜尋

當 `wiki/` 大到 `index.md` 撐不住時，加入本地搜尋引擎（純本機，資料不外流）。
